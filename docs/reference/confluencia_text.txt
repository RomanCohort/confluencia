架构不行？A: 不是架构问题。早期 Torch-Mamba 使用原始 mamba-ssm 配置，缺少多尺度池化和特征工程增强。2.0 版本引入 Mamba3Lite +四尺度池化 + k-mer哈希 + 生化统计的组合特征，已显著改善。HGB 在 160 样本上 R²=0.9996 的结果也验证了特征工程的有效性。Q: 为什么不用纯深度学习而要 MOE？A: 小样本场景下（<300样本），深度学习容易过拟合。MOE的自适应档位设计确保：少量样本用简单模型（Ridge+HGB），样本多了才加入 RF/MLP，兼顾偏差-方差权衡。Q: CTM 动力学有什么生物学依据？A: CTM 四房室模型（吸收→分布→效应→代谢）是经典药代动力学框架。我们的创新在于：将机器学习预测的 6 项微机制指标映射为CTM 参数，实现了”AI 预测 → 机制仿真”的闭环。RNACTM 更是首个面向 circRNA 的六房室模型，覆盖 LNP包裹→内吞→胞质释放→翻译→清除全链路。Q: 代理监督目标可靠吗？A: 代理目标不是替代真实标签，而是在无标签场景下提供可训练信号，使系统”不趴窝”。其设计原则是：利用可观测的物理量（剂量、频次、circRNA 表达量、IFN 评分）组合出与疗效正相关的代理信号。一旦有真实标签，系统自动切换到监督模式。Q: 数据量这么小，结果可信吗？A: 我们专门为小样本设计了多重保障：1. MOE 自适应档位避免过拟合2. 重复实验 + 95% CI 报告不确定性3. OOD 检测识别分布外样本4. 可靠性评估框架包含统计显著性检验5. 鲁棒性测试覆盖 20% 缺失值、极端异常值、极小数据集Q: 模型权重与大文件如何获取？如何校验完整性？A: 仓库的 build/目录包含示例权重(如、denseweight.h5)。建议将正式的大模型放到 release 或外部存储，并提供 scripts/download_models.ps1 / scripts/download_models.sh 及 SHA256 校验和。下载示例：powershell     powershell -ExecutionPolicy Bypass -File scripts\download_models.ps1校验示例：Get-FileHash .\\model.pt -Algorithm SHA256（PowerShell）或 sha256sum model.pt（Linux）。Q: 如果没有 RDKit，系统会报错吗？如何降级？A: 不会。代码采用懒加载与回退策略：若 RDKit 不可用，系统自动降级为字符/位置感知哈希指纹，保持基础功能。要启用 RDKit，请安装对应平台包并重启环境（参考confluencia-2.0-drug/requirements.txt）。Q: 如何安全地添加自定义 PDE 残差函数而不引入代码执行风险？A: 推荐做法：将残差函数写成独立 Python 模块并通过 importlib 以受控方式加载；避免在前端直接使用 exec。在生产环境应禁用或沙箱化前端上传功能（独立容器、资源限制、静态分析与白名单函数）。在注册前对 residual_fn 做静态/运行时校验和单元测试，注册接口为 MultiScaleModel.register_physics(residual_fn, coeff_fn)。Q: 推荐的计算资源与训练时间是多少？A: 依赖模型规模與数据量：经典 ML 在 CPU 上即可；Torch-Mamba 或大模型训练建议单卡 12–16GB GPU。示例估算：160 样本、40 epoch 的 Torch-Mamba 在 16GB GPU 上通常需数分钟到数十分钟；大规模预训练或进化优化可能需数小时至數天。Q: 如何运行 smoke tests 与 CI？A: 使用仓库内的 smoke 测试脚本：powershell     python .\\confluencia-2.0-epitope\\tests\\smoke_test.py     python .\\confluencia-2.0-drug\\tests\\smoke_test.py在 CI 中建议先安装 requirements-shared-minimal.txt 后执行上述脚本与 pytest。Q: 我想贡献代码，最佳实践与 PR 流程是什么？A: 建议流程：fork → 新分支（feat/<area>-<short>）→ 本地运行对应 smoke tests → 添加/更新单元测试 → 格式化（black/isort）→ 提交并开 PR，PR 描述请包含复现命令与测试结果。Q: 模型导出与反序列化安全注意事项？A: 优先使用安全格式（JSON + numpy 压缩权重或 torch  metadata.json）。避免发布任意 pickle。文档“6.4 模型安全导出”有示例导出格式。Q: 数据许可与敏感性如何处理？A: 外部数据请记录来源、检索语句与下载日期；受限数据需说明访问许可并提供脱敏或合成示例以供公开复现。将许可与字段说明写入 data/README.md。Q: 如何启用/调整不确定性与 OOD 检测？A: MOE 不确定性为专家预测标准差，可结合温度缩放、贝叶斯后处理或基于特征分布的 Mahalanobis/ODIN 方法增强。训练集中可保存每特征 5%/95% 分位数用于快速 OOD 判定。
